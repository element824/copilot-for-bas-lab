# Section 0 · Overview — Copilot for BA workflows

⏱ **10 minutes** · 🎯 You'll fork the lab repo and open it in your browser.

## What you'll do

1. Listen to the **3-minute pitch** ([PITCH.md](../../PITCH.md)).
2. **Fork** this lab repo to your own GitHub account.
3. Open your fork in the browser at `https://github.dev/<your-username>/copilot-for-bas-lab`.
4. React 👍 in the chat when you're done.

## Why we start with the wiki, not the editor

Your BAs are already experts in **page-centric** thinking — narrative, headings, ambient discovery. The trick is to **wrap GitHub in a wiki illusion** so you keep that muscle memory and gain two superpowers Confluence cannot give you:

1. The spec **lives next to the code** that implements it.
2. **Copilot** is your co-author — turning prose into acceptance criteria, generating templates, answering "where is X defined?".

## How to fork (step-by-step)

1. Go to the lab repo on github.com (the URL the facilitator shares).
2. Click the **Fork** button (top-right).
3. Choose **your own account** as the owner.
4. Click **Create fork**.
5. When it lands on your fork, look at the URL — it should be `github.com/<your-username>/copilot-for-bas-lab`.
6. Tap the `.` (dot) key. The page transforms into a browser-based VS Code.
