# Pre-flight checklist — please complete before the lab

> ⏱️ **5 minutes.** Doing this tonight saves you 15 minutes of frustration tomorrow morning.

The lab is hands-on. Each minute you spend setting up during the session is a minute you don't spend learning. Please tick everything below before you join.

---

## ✅ 1 · GitHub account

- [ ] You can sign in to **https://github.com** with your work credentials.
- [ ] When you click your profile picture (top-right), you see your name and account.

> If your organisation uses **GitHub Enterprise Cloud with SSO**, also confirm you can sign in via your corporate identity (you'll be redirected to your normal sign-in page).

## ✅ 2 · GitHub Copilot licence

- [ ] At https://github.com/settings/copilot you can see "**GitHub Copilot is active for this account**".
- [ ] If you don't, ask your IT / GitHub admin **today** — it can take a few hours to provision.

## ✅ 3 · VS Code installed

- [ ] You have **VS Code** installed on your laptop. (Download: https://code.visualstudio.com/)
- [ ] Open VS Code → bottom-left "👤 Accounts" icon → **sign in to GitHub**. You should see your profile name appear.
- [ ] In the Extensions panel (`Ctrl+Shift+X`), search for **GitHub Copilot** — confirm it's installed and shows "Disable / Uninstall" (i.e. it's already on).

> 💡 **If you cannot install VS Code** (locked-down laptop), don't panic. The lab also works in your browser at https://github.dev — but please test that you can open a repo there before the session.

## ✅ 4 · Open Copilot Chat — the 30-second self-test

This proves your environment is ready end-to-end.

1. In VS Code, click the **Copilot Chat** icon (it looks like a chat bubble) in the left sidebar — *or* press `Ctrl+Alt+I`.
2. Type: **`hello, are you ready for tomorrow?`** and press Enter.
3. You should see a friendly Copilot response within ~5 seconds.

- [ ] You got a response. ✅ You're ready.
- [ ] You got an error. ❌ See troubleshooting below.

## ✅ 5 · Browser fallback

If your VS Code can't connect tomorrow, you'll fall back to the browser. Test it now:

- [ ] Open https://github.dev in your browser. You should see a VS Code-like editor in your browser tab.

## ✅ 6 · Headset & second screen (if possible)

- [ ] Headset for audio (the facilitator is virtual).
- [ ] A second monitor is **strongly recommended** — Teams / Zoom on one, your VS Code on the other. Otherwise you'll be alt-tabbing every 30 seconds.

## ✅ 7 · Things you DO NOT need

- ❌ You do NOT need to install git, Node.js, Python, or any developer tool.
- ❌ You do NOT need to know what a "branch" or "commit" is — we'll use plain-English words.
- ❌ You do NOT need any prior coding experience.

---

## Troubleshooting

| Symptom | Fix |
|---|---|
| "Copilot is not signed in" | In VS Code, bottom-right status bar → click the Copilot icon → Sign in. |
| "You don't have access to Copilot" | Your licence isn't provisioned yet. Email your GitHub admin **today** with your GitHub username. |
| VS Code won't install (admin policy) | Use https://github.dev in your browser instead. |
| SSO redirect loop | Sign out of all GitHub sessions in your browser, close all tabs, sign in fresh. |
| "It works but it's slow" | Make sure you're on the corporate network, not split-VPN routing GitHub elsewhere. |

If none of these fix it: **email the facilitator before 5pm today** with a screenshot. We'll sort it together.
