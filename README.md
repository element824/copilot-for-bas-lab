# GitHub Copilot for Business Analysts — Hands-on Lab

> **A 90-minute hands-on lab for BAs.** By the end, you will have created your own GitHub project space, written a structured spec, refined it with Copilot, and shipped it through a pull request — all without writing a single line of code.

## Before the lab

> ⚠️ **Do this in the 24 hours before the session.** It takes 5 minutes and will save you an embarrassed "my Copilot won't load" moment in front of the room.

Open **[PRE-FLIGHT.md](./PRE-FLIGHT.md)** and run the checklist.

## What you'll build

By 90 minutes from now, **in your own forked repo**, you will have:

- ✅ A **project space** on GitHub (repo + Project board + first Issue)
- ✅ A **structured spec** (overview · personas · user stories · acceptance criteria · decisions)
- ✅ A **Copilot-refined user story** with Gherkin acceptance criteria
- ✅ A **pull request** showing your draft → review → merged change
- ✅ An **end-to-end trace**: Issue → Spec → Code reference → PR → Merge

You'll keep your fork. You can take it back to your team and start the trial Monday.

## How this lab works

- **You drive your own keyboard.** The facilitator demos each step, then you do it. Don't watch — *do*.
- **Each section ends with a checkpoint.** React with 👍 in the meeting chat when you're done so the room moves together.
- **Stuck? Don't suffer.** Drop a message in the lab chat; the in-room helper or the facilitator will unblock you. See [STUCK.md](./STUCK.md) for the top 5 issues and one-line fixes.
- **Everything is yours.** This repo is yours to fork, edit, break and rebuild. Nothing here is the customer's production data.

---

## What you take home

- Your **own forked repo** with your spec, Project, and PR.
- A **2-week trial plan** drafted in the final 5 minutes, ready to share with your manager.
- The **templates and Copilot prompts** you used today — re-usable for any spec.

---

> 💡 **First time here?** The lab assumes basic familiarity with what a requirement document is. No coding, no git, no command line knowledge required. The facilitator will use plain-English language throughout.
