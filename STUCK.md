# Stuck? Top 5 issues and one-line fixes

> Drop your symptom in the lab chat. The helper or facilitator will point you at the matching fix below.

## 1. "Copilot Chat shows no response / 'not authenticated'"

**Fix:**
1. Bottom-right status bar in VS Code → click the small Copilot icon.
2. Click **Sign in to GitHub**.
3. A browser tab opens → approve → return to VS Code.
4. Try the chat again.

If still failing in github.dev: click your profile picture (top-right) → **Sign out** → sign back in.

## 2. "I forked but I can't see my fork"

**Fix:**
- Go to https://github.com/&lt;your-username&gt;?tab=repositories — the fork should be listed.
- If not, the fork didn't complete. Go to the lab repo, click **Fork** again, choose your own account as the owner, untick "Copy the main branch only" if it's offered.

## 3. "I'm in github.dev but Copilot Chat isn't there"

**Fix:**
- github.dev has limited Copilot support. Two options:
  - **Codespaces:** click the green **Code** button → **Codespaces** → **Create codespace on main**. Full Copilot support.
  - **Local VS Code:** click **Code** → **Open with VS Code Desktop**. Sign in if prompted.

## 4. "I clicked Commit and it asked me about a branch"

**Fix:**
- This is the "draft copy" step. Just **accept the suggested name** and click **Commit**. Then GitHub will offer to create a Pull Request — say yes.
- Don't worry about the technical name. The helper will walk you through it if needed.

## 5. "The diff view looks scary / coloured / I don't understand it"

**Fix:**
- Look for a tab labelled **Rich diff** or the toggle that says **Source / Rendered**. Click **Rendered**.
- You'll now see your changes as the new version of the page, the way Confluence would show track changes. The colourful "code-style" view was the raw diff — you can ignore it.
